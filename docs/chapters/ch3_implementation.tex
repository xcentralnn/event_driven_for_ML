\section{Implementation}
\subsection{Overall System Architecture}
Figure \ref{fig:arch} (to be added) describes the overall system architecture, including the main components:
\begin{enumerate}
    \item \textbf{Kubernetes Cluster + Knative Serving:} The platform running serverless functions/services.
    \item \textbf{Prometheus:} Collects real-time metrics from Knative Autoscaler, Activator, Queue-Proxy.
    \item \textbf{ML Forecasting Service:} The workload forecasting inference service, receiving data from Prometheus, returning predicted values.
    \item \textbf{Predictive Scaler Controller:} The coordination component, reading forecast results and adjusting Knative's autoscaling configuration (e.g., via minScale or External Scaler/KEDA).
\end{enumerate}

The data flow operates in a cycle: Prometheus collects metrics $\rightarrow$ ML Service queries historical data and generates forecasts $\rightarrow$ Predictive Scaler Controller calculates the required number of pods according to formula (2.4) $\rightarrow$ Controller calls Kubernetes/Knative API to adjust the pod count.
